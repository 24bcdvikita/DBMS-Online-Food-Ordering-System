# DBMS-Online-Food-Ordering-System
Aim
The aim of this project is to design and develop a Database Management System for an Online Food Ordering System that allows customers to browse menus, place orders, and make payments online, while enabling restaurants and administrators to manage data efficiently. The system ensures accuracy, automation, and data integrity using relational database concepts and SQL.

Objectives
Relational Database Design: Create tables for Customers, Restaurants, MenuItems, Orders, and Payments with proper relationships and constraints.
Automation: Digitalize food ordering, billing, and payment processes to reduce manual errors.
Data Normalization: Apply up to 3NF to remove redundancy and ensure consistency.
SQL Implementation: Use SQL queries for data creation, manipulation, and reporting.
User Experience: Ensure fast and accurate data access for customers and administrators.

Scalability: Design a flexible database that can grow with more users and restaurants.

Learning Outcome: Gain practical experience in DBMS concepts, SQL, and real-world problem solving.
Tools & Technologies Used
Database: MySQL

Backend: SQL / PL-SQL

Frontend (Optional): HTML, CSS, PHP, or Python Flask
